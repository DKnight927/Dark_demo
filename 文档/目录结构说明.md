# 目录结构说明

## 根目录
- `自动加载/`：全局单例、全局服务、全局配置入口。
- `文档/`：策划、规则说明、上下文锚点、开发文档。
- `数据/`：定义资源、存档数据、内容资源。
- `场景/`：应用层、城镇域、地牢域、共用与调试场景。
- `脚本/`：应用逻辑、数据脚本、领域逻辑、表现脚本、工具脚本。
- `资源/`：占位图、正式美术、字体、音频、特效。
- `主题/`：Godot 主题与 UI 风格资源。
- `本地化/`：文本与多语言资源。
- `测试/`：测试脚本、测试资源、调试入口。

## 双域边界
- `场景/城镇域` 与 `脚本/领域/城镇域` 只负责局外逻辑。
- `场景/地牢域` 与 `脚本/领域/地牢域` 只负责局内探索与远征逻辑。
- 战斗规则核心逻辑统一收敛到 `脚本/领域/战斗`。

## 数据与脚本关系
- `数据/定义/` 主要承载 Godot 资源文件。
- `脚本/数据/` 主要承载这些资源对应的脚本类与运行时数据对象。
- 静态定义、持久化状态、运行时状态统一优先在 `脚本/数据/` 中声明。
