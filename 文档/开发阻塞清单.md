# 开发阻塞清单

这份清单只记录“会直接卡住后续开发”的问题，不记录一般性优化项。

## P0 阻塞

### 1. 正式资源目录尚未落地

- 现状：
  当前 `数据/` 只有演示数据，正式 `定义/` 资源目录此前不存在。
- 影响：
  无法开始系统化创建 `.tres`，资源命名、分类、引用路径都不稳定。
- 已处理：
  已建立 [`../数据/定义`](../数据/定义) 及子目录骨架。
- 后续动作：
  先创建首批状态、词缀、技能、英雄、敌人资源。

### 2. 缺少部分关键数据脚本

- 现状：
  之前文档引用了 `条件属性修正定义`、`随机事件定义`、`事件选项定义`、`特质定义`、`怪癖定义`、`疾病定义`，但仓库里未完全落地。
- 影响：
  资源创建无法闭环，事件、特质、疾病系统无法正式资源化。
- 已处理：
  已补齐对应脚本骨架。
- 后续动作：
  用这些类实际创建 `.tres`。

### 3. 场景与脚本缺少正式域骨架

- 现状：
  之前仓库几乎只有战斗调试场景和战斗域脚本。
- 影响：
  后续无法自然承载“主流程 -> 城镇 -> 地牢 -> 战斗 -> 结算”的串联开发。
- 已处理：
  已建立场景和脚本正式目录骨架。
- 后续动作：
  创建主流程壳层、城镇壳层、地牢壳层的基础场景和控制器。

## P1 重要缺口

### 4. 首发英雄 / 怪物大多仍是摘要版

- 现状：
  除剑圣和腐化领主外，多数英雄与怪物仍标记为“摘要，待扩展”。
- 影响：
  不适合作为最终资源源头，后续会反复返工。
- 后续动作：
  优先补齐首发 4 英雄和第一章怪物。

### 5. 特质 / 怪癖 / 疾病内容库不完整

- 现状：
  规则已完整，但内容条目此前还停在示例层。
- 已处理：
  已整理完整资源表。
- 后续动作：
  按表创建定义资源，并补入招募、结算、事件引用。

### 6. 路线图缺少验收标准

- 现状：
  旧路线图偏阶段描述。
- 已处理：
  已新增任务总表。
- 后续动作：
  后续开发应按任务总表推进，而不是只参考阶段名。

## 当前建议

- 先跑通第一章最小可玩资源包
- 再补城镇域 / 地牢域正式骨架
- 最后扩内容量，而不是先铺更多职业和怪物
