# 英雄文件模板

后续新增英雄，建议尽量按这个结构写。这样人和 agent 都更容易扫读、抽字段、做资源化。

## 推荐结构

### 1. 基本信息

- 职业名：
- 英雄ID：
- 主属性：
- 定位：
- 推荐站位：
- 内容状态：草案 / 已定稿 / 待平衡

### 2. 基础属性

- 初始血量：
- 初始魔力：
- 初始力量：
- 初始敏捷：
- 初始智力：
- 力量成长：
- 敏捷成长：
- 智力成长：
- 攻击附加最小值：
- 攻击附加最大值：
- 初始护甲：
- 基础行动速度：
- 初始暴击：
- 初始闪避：
- 初始命中：

### 3. 专属技能

- 技能名：
- 施放站位：
- 目标站位：
- 每级效果：
- 魔力消耗：
- 冷却：

### 4. 传承技能

- 技能名：
- 每级效果：

### 5. 专属天赋

- 天赋名：
- 触发条件：
- 效果：
- 限制：

### 6. 专属暗金饰品

- 饰品名：
- 固定属性：
- 独特效果：

### 7. 关联规则

- 依赖的系统规则：
- 若与公共规则冲突，按哪个文件为准：

## 写法建议

- 数值尽量写成可提取字段，不要只写自然语言段落
- 技能描述优先写“条件 + 效果 + 限制”
- 如果是摘要版，开头写明“内容状态：摘要”
