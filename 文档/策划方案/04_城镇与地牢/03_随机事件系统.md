11.6 随机事件完整规则

事件结构：
- 事件ID
- 标题
- 描述
- 权重
- 是否唯一
- 可出现层级
- 选项列表

选项结构：
- 文本
- 是否需要属性检定
- 检定属性
- 难度
- 成功结果
- 失败结果

难度标准：
- 简单：20
- 普通：35
- 困难：50
- 极难：65

属性检定规则：
- 从参战英雄中取对应属性最高者
- 若数值 > 难度则成功
- 若数值 <= 难度则失败

事件结果类型：
- 获得 / 失去金币
- 获得 / 失去补给
- 获得装备
- 获得技能书
- 恢复 / 损失 HP
- 恢复 / 增加压力
- 获得特质
- 获得怪癖
- 感染疾病
- 触发战斗
- 改变光照

唯一事件规则：
- 每战役只触发一次
- 记录在战役状态.已触发唯一事件

11.6.1 随机事件服务

脚本：`脚本/领域/地牢域/事件处理服务.gd`

职责：
- 按地牢层级、唯一性、前置条件筛选事件池
- 解析选项并进行属性检定
- 产出标准化事件结果

核心接口：
- `static func 抽取事件(地牢会话状态, 地牢定义, 事件池) -> 随机事件定义`
- `static func 获取可用选项(事件定义, 队伍快照) -> Array`
- `static func 结算事件选项(事件定义, 选项定义, 地牢会话状态, 战役状态) -> Dictionary`

标准结果结构：
- `文本日志`
- `获得资源`
- `失去资源`
- `状态变化`
- `HP变化`
- `压力变化`
- `触发战斗`
- `新增特质`
- `新增怪癖`
- `新增疾病`
