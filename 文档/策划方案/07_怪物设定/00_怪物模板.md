# 怪物文件模板

后续新增怪物，建议尽量按这个结构写。这样方便 agent 抽取词缀、层级、AI、掉落等信息。

- 怪物ID：
- 层级：
- 等级：
- 阵营 / 类型：
- 内容状态：草案 / 已定稿 / 待平衡

### 2. 属性

- 血量：
- 攻击：
- 护甲：
- 魔抗：
- 速度：
- 其他关键属性：

### 3. AI 与行为

- AI 类型：
- 主要目标选择逻辑：
- 技能优先级：

### 4. 技能

- 技能名：
- 施放条件：
- 效果：
- 冷却 / 频率限制：

### 5. 精英化 / 词缀规则

- 是否允许精英化：
- 精英化时词缀数量来源：
- 可抽词缀池：
- 是否存在签名词缀：

### 6. 掉落

- 金币：
- 材料：
- 装备：
- 特殊掉落：

### 7. 关联规则

- 依赖的系统规则：
- 若与公共规则冲突，按哪个文件为准：

## 写法建议

- 词缀数量不要在怪物文件里另起一套规则，应引用公共词缀系统
- Boss 要把“公共阶段规则”和“专属阶段技能”分开写
- 如果是摘要版，开头写明“内容状态：摘要”
