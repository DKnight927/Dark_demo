# 工程与 UI 导航

这个目录负责“怎么实现”和“怎么呈现”，更偏开发落地和流程衔接。

## 这个目录回答什么

- 存档和会话状态如何管理
- 信号和事件系统怎么组织
- UI 流程应该如何连接
- 开发路线图和内容推进顺序是什么

## 文件说明

- [`01_存档与会话管理.md`](./01_存档与会话管理.md)
  存档结构、序列化、会话状态边界。
- [`02_信号与事件系统.md`](./02_信号与事件系统.md)
  信号组织、跨模块事件传递。
- [`03_UI_UX流程规范.md`](./03_UI_UX流程规范.md)
  主要界面流程和交互规范。
- [`04_内容清单与实现路线图.md`](./04_内容清单与实现路线图.md)
  开发顺序、阶段目标、落地路线。

## 建议阅读顺序

1. [`04_内容清单与实现路线图.md`](./04_内容清单与实现路线图.md)
2. [`01_存档与会话管理.md`](./01_存档与会话管理.md)
3. [`02_信号与事件系统.md`](./02_信号与事件系统.md)
4. [`03_UI_UX流程规范.md`](./03_UI_UX流程规范.md)

## 常见修改场景

- 改开发顺序：优先改 `04`
- 改状态流转和保存：优先改 `01`
- 改模块通信：优先改 `02`
- 改界面路径：优先改 `03`

## 依赖关系

- 连接所有系统目录，是策划到实现的桥梁层
- 与 `脚本/`、`场景/` 和 `数据/` 的实际落地最直接相关
