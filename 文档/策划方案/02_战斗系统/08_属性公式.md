第七章：属性系统完整公式

7.1 主属性加成

- 每个职业有 1 个主属性
- 主属性在基础值上额外乘以 1.2
- 小数统一向下取整

7.2 英雄派生公式

生命值上限 = 初始血量 + 力量 × 20
魔力值上限 = 初始魔力 + 智力 × 5
物理攻击最小值 = 攻击附加最小值 + 力量 × 2
物理攻击最大值 = 攻击附加最大值 + 力量 × 2
魔法攻击 = 智力 × 3
护甲 = 初始护甲 + 敏捷 × 0.1%
魔法抗性 = 初始魔抗 + 智力 × 0.08%
速度 = 基础行动速度 + 敏捷 × 0.1
暴击率 = 初始暴击率 + 敏捷 × 0.1%
闪避率 = 初始闪避率 + 敏捷 × 0.1%

上限：
- 护甲上限 80%
- 魔抗上限 75%
- 暴击率上限 75%
- 闪避率上限 75%

7.3 升阶加成

- 每阶获得 10 点自由属性
- 最终伤害加成每阶 +2%
- 最终伤害减免每阶 +2%
- 升阶加成先计入主属性修正，再参与派生计算

7.4 装备 / 天赋 / 特质 / 怪癖 / 疾病叠加顺序

1. 英雄基础值
2. 等级成长
3. 升阶自由点
4. 主属性 1.2 倍
5. 装备固定属性
6. 装备随机属性
7. 天赋修正
8. 特质修正
9. 怪癖修正
10. 疾病修正

7.5 属性计算服务

脚本：`脚本/领域/共用/属性计算服务.gd`

职责：
- 根据英雄定义、等级、升阶、装备、天赋、特质、怪癖、疾病生成最终属性快照
- 计算战斗中实时属性
- 处理条件属性修正

核心接口：
- `static func 计算英雄城镇面板(英雄战役状态, 英雄定义, 数据仓库) -> Dictionary`
- `static func 构建战斗属性快照(英雄战役状态, 英雄定义, 数据仓库) -> Dictionary`
- `static func 构建敌人属性快照(敌人定义, 已附加词缀列表) -> Dictionary`
- `static func 获取实时属性(单位运行时, 属性: 属性类型, 战斗状态) -> float`

实现约束：
- 条件修正统一在 `获取实时属性` 时判定
- 快照中保存基础值与固定修正，条件修正不固化到快照
- 所有百分比修正最终统一转成乘算链
