# 基础总览导航

这个目录定义项目的基础规则边界，是其他所有策划目录的前置上下文。

## 这个目录回答什么

- 项目到底想做成什么
- 世界观和叙事框架是什么
- 工程目录和架构怎么约束
- 数据层应该有哪些核心结构

## 文件说明

- [`01_项目概览与核心原则.md`](./01_项目概览与核心原则.md)
  项目目标、核心体验、全局原则。
- [`02_世界观与叙事框架.md`](./02_世界观与叙事框架.md)
  世界背景、风格基调、叙事边界。
- [`03_目录结构与架构规范.md`](./03_目录结构与架构规范.md)
  代码目录、职责划分、模块边界。
- [`04_数据层完整规范.md`](./04_数据层完整规范.md)
  资源类、状态类、服务类所依赖的数据结构。

## 建议阅读顺序

1. [`01_项目概览与核心原则.md`](./01_项目概览与核心原则.md)
2. [`04_数据层完整规范.md`](./04_数据层完整规范.md)
3. [`03_目录结构与架构规范.md`](./03_目录结构与架构规范.md)
4. [`02_世界观与叙事框架.md`](./02_世界观与叙事框架.md)

## 常见修改场景

- 改项目范围或核心体验：优先改 `01`
- 改数据结构或资源字段：优先改 `04`
- 改代码架构边界：优先改 `03`
- 改世界观和叙事口径：优先改 `02`

## 依赖关系

- 战斗系统依赖这里的数据层定义
- 城镇、地牢、成长系统依赖这里的全局规则口径
- 工程实现目录应与这里的架构规范保持一致
