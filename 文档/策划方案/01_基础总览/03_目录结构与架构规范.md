3.2 三层数据模型

静态定义：
- Resource 资源
- 只读
- 描述模板和规则

持久状态：
- 存档序列化对象
- 表示战役长期进度

运行时状态：
- 战斗或探索会话中的临时对象
- 会话结束后写回持久状态

3.3 服务 / 控制器 / UI 边界

服务层：
- 只做计算和规则判断
- 不依赖场景树

控制器层：
- 持有当前域状态
- 按流程调用服务
- 负责发信号

UI 层：
- 不直接改状态对象
- 不直接进行跨系统调用
- 只展示状态、采集输入、响应事件

3.4 统一职责修正

为消除原稿冲突，现统一如下：
- 驿站：招募、刷新候选、英雄卡管理

- 仓库：扩容
- 领主居所：章节进度、统计、里程碑展示
