# Agent 开工说明

这份文件是给后续接手仓库的 agent 准备的。目标不是解释世界观，而是让 agent 尽快进入“安全可修改”状态。

## 第一步先读什么

1. [`README.md`](../../../README.md)
2. [`../../README.md`](../../README.md)
3. [`./README.md`](./README.md)
4. [`../01_基础总览/04_数据层完整规范.md`](../01_基础总览/04_数据层完整规范.md)
5. 如果任务和战斗相关，再读 [`../02_战斗系统/README.md`](../02_战斗系统/README.md)
6. 如果任务和职业相关，再读 [`../06_英雄设定/README.md`](../06_英雄设定/README.md)
7. 如果任务和怪物相关，再读 [`../07_怪物设定/README.md`](../07_怪物设定/README.md)

## 工作原则

- 先判断这是“系统规则修改”还是“单个内容修改”
- 系统规则修改：优先改基础总览、战斗系统、成长与装备、城镇与地牢等系统目录
- 单个内容修改：优先改英雄或怪物单文件，不要把个体设定写回系统目录
- 如果正文和附录冲突，以正文为准
- 如果单个英雄 / 怪物文件和系统规则冲突，以系统规则为准，再回头修单文件

## 常见任务怎么落点

- 改战斗流程：[`../02_战斗系统`](../02_战斗系统/)
- 改尸体、位移、伤害、Boss、词缀：[`../02_战斗系统`](../02_战斗系统/)
- 改装备词条或成长：[`../03_成长与装备`](../03_成长与装备/)
- 改城镇设施、地牢探索、事件：[`../04_城镇与地牢`](../04_城镇与地牢/)
- 改工程落地方式：[`../05_工程与UI`](../05_工程与UI/)
- 改职业：[`../06_英雄设定`](../06_英雄设定/)
- 改怪物：[`../07_怪物设定`](../07_怪物设定/)

## 修改前要先确认的事

- 这个改动是“全局规则”还是“个别条目”
- 有没有对应的上层 README 已经说明依赖关系
- 有没有需要同步修改的资源表、索引页、关联目录 README

## 修改后建议补什么

- 如果新增了新系统目录，补一个 `README`
- 如果新增了新英雄或怪物，尽量按模板写
- 如果改了规则源头，必要时更新 [`规则源头说明.md`](./规则源头说明.md)
