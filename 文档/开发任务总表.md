# 开发任务总表

这份文件把当前阶段需要做的事改写成更适合执行和协作的任务板。

## T1 资源基础层

- 任务：
  建立首批 `状态效果 / 怪物词缀 / 技能 / 英雄 / 敌人` 资源。
- 依赖：
  数据脚本骨架齐全，资源目录存在。
- 完成标准：
  第一章最小可玩资源包能在 Godot 中被创建并加载。

## T2 战斗资源化

- 任务：
  用正式资源替换演示战斗数据。
- 依赖：
  T1。
- 完成标准：
  调试战斗不再依赖硬编码演示工厂即可跑通。

## T3 首发阵容定稿

- 任务：
  将 `剑圣 / 盾卫 / 弓手 / 法师` 补齐到可直接建资源的完整规格。
- 依赖：
  英雄模板、成长规则。
- 完成标准：
  每个职业都具备明确的基础属性、两组技能、天赋、暗金饰品和推荐站位。

## T4 第一章怪物定稿

- 任务：
  将 `腐化民兵 / 腐化祭司 / 腐化骑士长 / 腐化领主` 补齐到可直接建资源的完整规格。
- 依赖：
  怪物模板、词缀系统、Boss 阶段规则。
- 完成标准：
  第一章敌人与 Boss 可直接建资源并形成完整战斗链。

## T5 特质 / 怪癖 / 疾病资源化

- 任务：
  按完整资源表建立定义资源，并接入招募、事件、远征结算。
- 依赖：
  `特质定义.gd`、`怪癖定义.gd`、`疾病定义.gd`。
- 完成标准：
  能在英雄持久状态中新增、移除、锁定并正确生效。

## T6 主流程骨架

- 任务：
  建立应用根、战役流程控制器、城镇控制器、地牢控制器、存档管理器。
- 依赖：
  正式脚本目录骨架。
- 完成标准：
  项目能表达“启动 -> 进入战役 -> 进入城镇 -> 进入地牢 -> 返回结算”的基本流转。

## T7 场景骨架

- 任务：
  建立主菜单、战役壳层、城镇壳层、地牢壳层场景。
- 依赖：
  T6。
- 完成标准：
  除战斗调试场景外，项目拥有正式主流程场景树。

## T8 第一章最小可玩

- 任务：
  跑通“4 英雄 + 4 敌人 + 1 Boss + 1 地牢 + 6 事件”的第一章最小闭环。
- 依赖：
  T1-T7。
- 完成标准：
  玩家可从主流程进入一轮完整远征并返回。

## 当前推荐执行顺序

1. T1
2. T2
3. T3 + T4
4. T5
5. T6 + T7
6. T8
