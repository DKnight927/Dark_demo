# 开工路径与首批脚本清单

## 当前开工目标
- 先完成工程骨架冻结。
- 先落“数据模型优先”的第一批脚本，而不是先堆 UI。
- 从第一版开始为以下热更新需求预留结构：`行动条 / 天赋 / 三维属性成长 / 怪物词缀`。

1. 冻结中文目录结构与基础工程命名。
2. 创建定义层、持久化层、运行时层的第一批基础脚本。
3. 完成英雄、技能、装备、怪物、天赋相关基础数据模型。
4. 完成属性成长与战斗单位投影。
5. 完成战斗运行时对象与行动链骨架。
6. 再进入战斗根场景与基础行动条系统。

## 关键设计提醒

### 行动系统
- 不使用传统固定回合制作为中间过渡方案。
- 第一版战斗排程就按“行动条 / 跑圈”思路设计。
- 速度应直接作用于行动进度推进，而不是仅用于每回合排序。

### 英雄成长
- 英雄必须具备主属性类型区分：`力量 / 敏捷 / 智力`。
- 英雄定义中需要有基础属性、成长值、专属技能、可学技能池、可用天赋信息。

### 装备系统
- 装备必须区分“定义”和“实例”。
- 随机词条和随机数值都只能存在于实例层，不能写死在定义层。

### 敌人与精英词缀
- 敌人定义应允许挂接词缀池或词缀接口。
- 怪物词缀设计要能影响属性、抗性、附加行为或规则豁免。

## 当前阶段性交付标准
- 工程目录完整且稳定。
- 首批脚本名单明确。
- 后续创建文件时不再临时决定放置位置与命名方式。
