# 上下文锚点

## 会话目标
- 项目目标是制作一个以最终商业成品为目标的 2D 回合制 RPG。
- 核心参考对象是《暗黑地牢》的系统复杂度与设计哲学，但不直接照搬实现。
- 技术路线为 `Godot 4.x + GDScript 2.0 + 数据驱动 + 状态机分层`。
- 项目要求长期协作中使用可持续的上下文压缩方式，避免遗忘关键决策、文件轨迹与下一步计划。

## 项目约束
- 整个项目采用中文目录、中文文件、中文资源命名。
- 系统分为 `城镇域` 与 `地牢域` 两个核心域，职责必须严格分离。
- 战斗、探索、城镇、存档等系统都需要从第一版开始为后续扩展预留接口。
- 前期允许使用占位资源，但布局、逻辑、规则不能依赖素材原始尺寸。

## 已确认设计
- 战斗基础空间规则采用双方各 `1-4 号位` 的逻辑槽位系统。
- 项目数据分为三层：`静态定义层`、`持久化状态层`、`运行时状态层`。
- 工程起步顺序是先建中文目录结构，再建核心 Resource 数据模型，再进入战斗骨架与地牢、城镇系统。
- 项目当前不是以临时 MVP 为目标，而是以生产级结构为目标推进。

## 热更新决策
- 行动顺序从“固定回合内重新排序”调整为“跑圈 / 行动条”模型。
- 英雄系统除技能外，还需要天赋系统。
- 属性系统细化为 `力量 / 敏捷 / 智力` 英雄，并具备固定成长值与派生属性关系。
- 怪物需要词缀系统，尤其是精英怪强化机制，如免疫、法力燃烧、力量强化等。
