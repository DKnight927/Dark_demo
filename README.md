# 黑暗之源（Dark Source）完整策划与实现指南

> 版本：v2.0 综合版  
> 引擎：Godot 4.6 · GDScript 2.0  
> 目标：直接指导项目从结构搭建到核心系统实现

## 项目概览

《黑暗之源》是一款 2D 回合制地牢爬行 RPG，核心体验参考《暗黑地牢》，强调资源紧缺、压力系统、站位策略与永久性成长/失去。

项目的设计目标不是做一个短平快的原型，而是从第一天起就按“完整商业成品”的标准来拆分模块、定义数据、规划实现顺序。

### 核心体验

- 局内：高压、高风险、强策略性的地牢探索与回合制战斗
- 局外：围绕英雄名册、设施投资、资源分配的长期经营与恢复循环
- 整体：角色不是绝对服从的棋子，而是会承受压力、产生缺陷、形成长期成长与创伤的“人”

## 开发总原则

### 1. 双域严格拆分

整个项目拆成两个核心域，互不混写：

- `城镇域`
  负责局外运营、英雄名册、设施、资源池、编队、远征结算回写
- `地牢域`
  负责局内探索、光照、遭遇、战斗、压力、掉落、撤退、任务推进

### 2. 数据驱动与状态机并重

- 局外系统以 `界面 + 数据驱动 + 持久化修改` 为核心
- 局内系统以 `状态机 + 规则解析 + 逻辑驱动` 为核心
- 复杂机制优先通过 `资源定义 + 运行时状态 + 拦截链 / 触发器` 扩展

### 3. 可扩展优先

底层数据结构从第一版开始就要为这些机制预留接口：

- 压力
- 折磨与美德
- 怪癖
- 疾病
- Buff / Debuff
- DOT / HOT
- 饰品
- 光照修正
- 死亡之门
- 尸体位
- 被动触发器

### 4. 美术占位与正式美术兼容

前期将使用 AI 生成的占位图，后期替换为正式像素美术。因此：

- 不允许把尺寸写死到 UI 布局逻辑中
- 不允许让贴图原始尺寸决定交互规则
- `Control` 界面优先使用容器、自适应布局、相对锚点、动态缩放
- 战斗站位逻辑只认“逻辑槽位”，不认屏幕像素位置

## 核心产品结构

### 三层主流程

- 应用层
  负责启动、菜单、档案载入、状态切换、全局服务
- 城镇域
  负责局外运营与长期状态管理
- 地牢域
  负责一次远征中的探索、战斗与会话状态

主流程如下：

```text
启动 -> 主菜单 -> 战役壳层 -> 城镇 -> 出征准备 -> 地牢探索 -> 战斗 / 事件 -> 远征结算 -> 返回城镇
```

### 战役壳层

战役壳层是项目的总流程容器，负责：

- 持有当前 `战役状态`
- 在城镇域与地牢域之间切换
- 接收 `远征结果` 并回写
- 生成下一次 `出征准备`

城镇域与地牢域并列挂载，不彼此嵌套。

## 数据模型

项目采用三层数据模型：

### 1. 静态定义（Definition）

- 只读，运行期间不改变
- 存储于 `数据/` 目录下的 `.tres` 文件
- 脚本继承 `Resource`，使用 `@export` 暴露属性

### 2. 持久状态（Campaign State）

- 随存档保存，跨关卡保留
- 存储于存档管理器的 JSON 序列化
- 脚本使用普通 class，提供 `to_dict()` / `from_dict()` 方法

### 3. 运行时状态（Runtime State）

- 仅存在于一次战斗/探索会话中
- 会话结束后将结果写回持久状态
- 脚本使用普通 class，不需要序列化

## 核心系统

### 站位系统

战斗双方各拥有 4 个逻辑战斗位。技能可用性和目标合法性严格由站位决定。

每个技能至少包含三类约束：

- 施法者可用站位
- 目标可选站位
- 目标阵营与目标规则

这个系统的价值是把职业、技能、敌人、位移、击退、拉拽统一到一套空间规则里。

### 回合排程

战斗回合采用“每轮重新生成行动顺序”的离散模型：

1. 每轮开始收集所有可行动单位
2. 计算行动值
3. 按行动值排序
4. 依次行动
5. 全员行动后开启下一轮

### 压力系统

压力是整个项目的核心戏剧张力来源：

- 低压力阶段：稳定推进
- 中压力阶段：开始出现负面反馈
- 高压力阶段：触发失控、崩溃或极端行为

### 战斗控制器与服务层

服务脚本只做规则计算，不持有状态；控制器持有状态并调度服务。

服务层规则：

1. 不继承 `Node`
2. 不持有状态
3. 方法命名以 `动词_名词` 为主
4. 除了修改传入的状态对象，不操作外部对象

## 实现路线图

### 第一阶段

- 搭建自动加载与全局事件总线
- 完成基础存档结构
- 打通战役壳层与城镇/地牢切换

### 第二阶段

- 完成战斗核心链路
- 行动顺序、站位合法性、目标解析、伤害/防御/命中结算
- 形成可运行的演示战斗闭环

### 第三阶段

- 压力、状态效果、掉落、死亡与撤退规则
- 把战斗结果写回持久状态

### 第四阶段

- 城镇设施、英雄养成、资源循环
- 完整远征体验与长期成长循环
